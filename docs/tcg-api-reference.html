TCG API endpoint reference

A working summary of the endpoints the Card Trading Engine uses to look up sets, cards, and market prices. Compiled from the official docs at tcgapi.dev/docs after the original scaffold's guessed endpoint shapes turned out not to match the live API.

Authentication & request basics

Every request requires an API key passed as a header — not a query parameter. All list responses share the same envelope shape.

API surface

Endpoints we rely on

The five endpoint groups relevant to the Deal Finder's card lookup, watchlist pricing, and dashboard trend features.

Sets

01
GET /sets?game=&page=&per_page=
List sets for a game. Returns id, name, slug, abbreviation, release_date, card_count, game_name. Sort client-side by release_date — the API returns them unordered.
List Sets
02
GET /sets/:id
Single set by internal numeric ID — same fields as the list endpoint plus last_synced_at.
Get Set
03
GET /sets/:id/cards?page=&per_page=
The official "cards in a set" endpoint — paginated, no free-text query. This is what powers the card lookup dropdown (the scaffold's original /cards?game=&set=&q= endpoint doesn't exist).
Cards in Set
04
GET /sets/:id/prices Pro tier
Bulk prices for every card in a set — useful for set-value tracking. Not used in Phase 1, but worth knowing about for future watchlist features.

Cards

05
GET /cards/:id
Card metadata by internal ID — name, number, rarity, image, tcgplayer_id, and custom_attributes (Pokémon: stage, HP, energy type, attacks, etc).
Card Detail
06
GET /cards/tcgplayer/:tcgplayerId
Look up a card by its TCGPlayer product ID. Returns the card plus a full prices[] array — one entry per printing (Normal, Foil, etc).
By TCGPlayer ID
07
GET /cards/:id/prices?printing=
The authoritative price lookup for a known card. Returns market_price, low_price, median_price, lowest_with_shipping, buylist_price, plus 24h/7d/30d % change. This is what feeds the deal algorithm's tcgMarket input.
Card Prices
08
GET /cards/:id/history Pro  /  /history/detailed Business
Weekly price history (from March 2025) with market_price, avg_sales_price, sales_volume per date — would power richer trend charts than what we can build from our own scan snapshots alone.

Search & Prices

09
GET /search?q=&game=&set_id=&rarity=&min_price=&max_price=&sort=
The real full-text search endpoint (q requires 2+ characters). Returns cards with pricing embedded inline in one call — closer to the original "type to search" UX than the dropdown-only approach. Sort options: relevance, price_asc, price_desc, name.
Full-text Search
10
GET /prices/top-movers?game=&direction=&period=&limit=
Cards with the biggest price swings (24h/7d/30d, up or down). Could power a "market movers" widget on the dashboard beyond what scan-result trends alone show.
Top Movers
Data shapes

Response envelope & key fields

Every list endpoint wraps results the same way — this is the part the original scaffold got wrong (it expected { sets: [...] } / { cards: [...] } instead of a generic data array).

Scaffold assumed
{ "sets": [...] }
{ "cards": [...] }
{ "prices": {...} }
Auth via x-api-key (lowercase, loosely placed)
Free-text search at /cards?q=
Actual API
{ "data": [...], "meta": {...}, "rate_limit": {...} }
Same data envelope on every list endpoint
Per-printing prices[] array on card-detail responses
Auth via X-API-Key header (capitalized, required)
Free-text search lives at /search?q= instead

Sealed products (booster boxes, tins, ETBs, cases) are returned through the same endpoints — identified by product_type: "Sealed Products" and condition: "Unopened" on listings. Filter with type=Sealed+Products on search/set endpoints if the watchlist ever expands beyond singles.

Applied to this project

How this maps onto the Card Trading Engine

What changed in lib/tcg.ts once the real endpoint shapes were confirmed against the live API.

01
getSets()
Now hits /sets?game=, unwraps data[], maps slug → code, and sorts client-side by release_date (newest first) since the API returns sets unordered.
02
getCardsInSet()
Switched from the non-existent /cards?game=&set=&q= to /sets/:id/cards, merging multiple printing rows per card into one entry with market/foil prices split out.
03
searchCards()
New helper that calls the real /search?q=&game=&set_id= endpoint — available if we want to bring back live free-text search instead of (or alongside) the set-scoped dropdown.
04
getCardPrice()
Now calls /cards/:id/prices directly — the authoritative per-card price source — and prefers the Normal printing as the deal algorithm's tcgMarket baseline.
05
Auth header fix
Changed from x-api-key to the documented X-API-Key — likely the actual root cause of "queries return no data" before any shape mismatches even mattered.
06
Sports Cards removed
Dropped from GAMES, GAME_MAP, and mock data — TCG API has no sports card coverage, so the category was undeliverable as scoped.

Still to verify: the exact field names returned by /sets/:id/cards and /search for card pricing (market_price vs. nested prices[]) — the current mapping in lib/tcg.ts handles both shapes defensively, but should be confirmed against a live response and tightened once you've run a real lookup.
